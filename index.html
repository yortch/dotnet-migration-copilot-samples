Getting Started

Clone the Sample Project

Start with the ContosoUniversity sample — a realistic legacy .NET application with common modernization challenges.

Step 1 — Clone or Fork the Sample Repository Terminal
1
Clone the repo git clone https://github.com/Azure-Samples/dotnet-migration-copilot-samples.git
2
Open the ContosoUniversity solution cd dotnet-migration-copilot-samples/ContosoUniversity

🏗️ Legacy Tech Stack

  • ASP.NET MVC 5 — ~10 years old (2014)
  • .NET Framework 4.8 — legacy, Windows-only
  • Entity Framework 6 — ~11 years old (2013)

⚙️ Infrastructure Dependencies

  • SQL Server LocalDB — local dev database
  • IIS / IIS Express — ~25 years old (1995)
  • MSMQ — ~28 years old (1997), deprecated

🎯 Upgrade Target

  • .NET 10 — latest, cross-platform
  • ASP.NET Core MVC — modern web stack
  • EF Core — modern ORM

Sample prompt to use with any option:

"Upgrade my project to .NET 10"

Setup & Run

Four Ways to Modernize

Each option provides complete Assess → Plan → Execute capabilities with setup instructions below.

1

IDE — Visual Studio & VS Code

Interactive, chat-driven modernization right in your editor with step-by-step AI guidance.

✅ Best When

  • Working on a specific project interactively
  • Want step-by-step review of each transformation
  • Prefer IDE-integrated, visual workflow
Setup Instructions IDE
1
Visual Studio VS 2022 (17.14.17+) or VS 2026 — modernization is built-in, no extra install needed.
2
VS Code Install GitHub Copilot modernization extension
3
Select Custom Agent Open Copilot Chat → select modernize-dotnet custom agent
4
Run Upgrade Enter prompt: Upgrade to .NET 10
2

Copilot CLI

Same modernize-dotnet agent, terminal-first experience — no IDE required.

✅ Best When

  • Terminal-first or SSH-based development
  • Quick assessments from the command line
  • Scripting and lightweight automation
Setup Instructions CLI
1
Launch Copilot CLI Start the GitHub Copilot CLI in your project directory
2
Install Plugin Marketplace Run: /plugin marketplace add dotnet/modernize-dotnet
3
Select Agent Run: /agent and select modernize-dotnet
4
Run Upgrade Enter prompt: Upgrade to .NET 10 — review generated plan, then let the agent execute upgrades.
3

GitHub Copilot Cloud Agent

Autonomous agent that runs on GitHub, creates branches and PRs for your team to review.

✅ Best When

  • Want fully autonomous, hands-off upgrades
  • Prefer GitHub-native PR review workflow
  • Delegating modernization to the cloud
Setup Instructions GitHub.com
1
Configure Agent Follow setup at github.com/dotnet/modernize-dotnet/coding-agent
2
Add copilot-setup-steps.yml Configure the Cloud Agent workflow in your repository's .github/workflows/ directory
3
Start Cloud Agent Session Go to the Agents tab of your GitHub project, select modernize-dotnet from the custom agents dropdown, enter prompt Upgrade project to .NET 10, and optionally change the model.
4
Review & Submit Review committed code and submit a PR when done.
4

Modernize CLI (Public Preview)

Batch assessment and upgrade engine purpose-built for portfolios at scale with CI/CD integration.

✅ Best When

  • Assessing or upgrading dozens/hundreds of repos
  • CI/CD pipeline integration needed
  • Enterprise-wide portfolio modernization
Setup Instructions CLI / CI/CD
1
Install Modernize CLI Follow the installation guide at aka.ms/ghcp-modernization-agent
2
Batch Assessment Point the CLI at multiple repos or a portfolio manifest to scan for modernization candidates.
3
Define Standards Configure org-wide upgrade skills and patterns for consistent outcomes across all projects.
4
Execute & Integrate Run upgrades in semi or fully automated mode. Integrate into GitHub Actions or ADO pipelines.
